Especificaciones Técnicas

1. Introducción

STC Cloud es una plataforma de gestión de impresión de clase empresarial. El despliegue se realiza mediante micro-agentes ligeros (STC Cloud Monitor) que se instalan en la infraestructura del cliente para recolectar telemetría de dispositivos de red mediante protocolos estándar.

Este documento detalla los requisitos técnicos mínimos que el administrador de IT debe asegurar antes del despliegue del agente para garantizar una operación continua y segura.

2. Sistemas Operativos

  • Windows Server 2016 / 2019 / 2022 (Todas las ediciones)
  • Windows 10 / 11 Pro / Enterprise (64-bit)

Nota: El agente se ejecuta como un Servicio de Windows ("STCCloudMonitor") bajo la cuenta LocalSystem. Es totalmente compatible con entornos virtualizados (VMware, Hyper-V, Azure, AWS).

Configuración de Red

4. Requerimientos de Red y Firewall

El agente utiliza una arquitectura Zero-Inbound. No requiere ninguna apertura de puertos de entrada desde Internet hacia la red del cliente.

Conectividad Externa (Hacia la Nube)

Puerto Protocolo Destino Función
443 TCP (HTTPS) stc-cloud-api.onrender.com Telemetría, Activación y Heartbeat
443 TCP (WSS) stc-cloud-api.onrender.com Canal de control en tiempo real (WebSockets)
443 TCP (HTTPS) github.com / raw.githubusercontent.com Descarga de actualizaciones firmadas

Conectividad Interna (LAN)

Puerto Protocolo Función
161 UDP (SNMP) Lectura de contadores y estados de impresoras
9100 / 80 / 443 TCP Pre-verificación opcional de alcanzabilidad
Arquitectura de Seguridad: Todas las comunicaciones externas están cifradas mediante TLS 1.2 o superior. El agente utiliza Exponential Backoff para reconexiones automáticas en caso de inestabilidad de red.

Proxy y Despliegue

5. Soporte de Proxy Corporativo

En redes con salida restringida, el agente admite configuración de Proxy HTTP/HTTPS con soporte para autenticación básica.

  • Túnel CONNECT: Soporta el establecimiento de túneles para tráfico HTTPS y WebSockets (WSS).
  • Persistencia: Las credenciales del proxy se almacenan de forma segura en la configuración cifrada del agente.
  • Cuentas de Servicio: Se recomienda el uso de una cuenta de servicio con contraseña que no expire para evitar interrupciones de monitoreo.

6. Protocolo de Verificación de Integridad (SHA-256)

Verificación de Seguridad: El software STC Cloud Monitor se distribuye como un binario nativo independiente. Para garantizar la máxima seguridad y confianza en entornos corporativos, se proporciona una firma de hash SHA-256 oficial para cada versión. Este método de verificación asegura que el instalador no ha sido alterado por terceros durante la transferencia y es el estándar recomendado para auditorías de infraestructura.

Verificación de Hash SHA-256

El paquete de distribución incluye el script verify_hash.ps1. Ejecutar desde PowerShell:

.\verify_hash.ps1 -FilePath "STC-Monitor-Setup.exe" -ExpectedHash "<HASH_OFICIAL_STC>"

El hash SHA-256 oficial de cada versión se publica en el portal STC Cloud y en las notas de lanzamiento. Para gestión de alertas de SmartScreen/Defender, consultar la sección 9 del Manual del Administrador IT v1.5.

7. Instalación y Despliegue Masivo

Para entornos con múltiples sites, el instalador soporta parámetros de línea de comandos para despliegues desatendidos vía GPO, Microsoft Intune o scripts de PowerShell.

Ejemplo de Instalación Silenciosa

STC-Monitor-Setup.exe /VERYSILENT /KEY=TU_CLAVE_DE_ACTIVACION /SERVER=https://stc-cloud-api.onrender.com

Directorio de Datos

El agente almacena su configuración cifrada y base de datos local en:

C:\ProgramData\STCCloudMonitor\
