Auditoría de Privacidad · Versión 1.5

Conclusión Ejecutiva: El agente STC Cloud Monitor recopila exclusivamente métricas operativas de impresoras (contadores de páginas, fabricante, modelo, número de serie). No recopila, procesa ni transmite ningún dato de usuarios, documentos, credenciales de red ni Información de Identificación Personal (PII). Todos los datos de configuración se almacenan cifrados con AES-256-GCM ligado al hardware del equipo.

1. Datos Recopilados vía SNMP de Impresoras

El agente escanea rangos IP configurados por el administrador IT y consulta impresoras via SNMP (UDP puerto 161, lectura solamente). La siguiente tabla detalla cada campo recopilado:

# Campo Tipo Origen (OID SNMP) Propósito Sensibilidad ¿Se transmite?
1 ip String IPv4 Dirección de destino del scan Identificar la impresora en la red del cliente Media Sí
2 brand Enum sysObjectID (1.3.6.1.2.1.1.2.0) Selección de OIDs correctos para la marca Baja Sí
3 model String (máx. 100 chars) sysDescr — truncado Identificación del modelo para reportes Baja Sí
4 sysDescr String (máx. 255 chars) 1.3.6.1.2.1.1.1.0 Descripción técnica del dispositivo Baja Solo local
5 sysName String 1.3.6.1.2.1.1.5.0 Nombre del dispositivo en la red Baja Solo local
6 serial String OID específico por fabricante Identificador único del dispositivo físico (usado como device_id) Media Sí
7 total_pages Integer OID contador por marca Facturación de páginas totales impresas Baja Sí
8 mono_pages Integer OID contador por marca Facturación de páginas monocromo Baja Sí
9 color_pages Integer OID contador por marca Facturación de páginas color Baja Sí
10 time ISO 8601 Reloj del sistema Windows Timestamp de la lectura SNMP Baja Sí

Nota técnica SNMP: El agente realiza consultas SNMP de solo lectura (GET). No utiliza SetRequest ni operaciones de escritura. El protocolo opera en UDP puerto 161 sobre la red local del cliente. No existe tráfico SNMP saliente a Internet.

2. Payload Transmitido al Servidor STC

El endpoint receptor es POST {serverUrl}/api/v1/devices/sync. La transmisión usa HTTPS (TLS 1.2+) con autenticación Bearer JWT. El payload contiene exactamente estos campos:

Campo Contenido Sensibilidad
device_id Número de serie de la impresora; si no disponible, su IP Media
ip Dirección IP local de la impresora (no ruteable externamente) Media
brand Fabricante detectado (hp / lexmark / samsung / ricoh / brother / xerox / generic) Baja
model Modelo del dispositivo (string, máx. 100 chars) Baja
time Timestamp ISO 8601 de la lectura Baja
total_pages Contador entero de páginas totales Baja
mono_pages Contador entero de páginas monocromo Baja
color_pages Contador entero de páginas color Baja
offline Flag booleano, siempre true (constante hardcoded) Baja

3. Almacenamiento Local

Ubicación: C:\ProgramData\STCCloudMonitor\ — accesible solo por la cuenta de servicio Windows.

Tabla / Archivo Campos almacenados Retención Propósito
local.db → readings_queue device_id, ip, brand, model, time, total_pages, mono_pages, color_pages, synced, created_at 7 días (purga automática) Cola de envío con soporte offline. Datos se eliminan tras sincronización o vencimiento.
local.db → known_devices ip, serial, brand, model, registered, last_seen Catálogo persistente Evitar re-registro de dispositivos ya conocidos. No contiene contadores.
config.enc Ver Sección 4 (cifrado AES-256-GCM) Indefinida Configuración operativa del agente.
agent.log Eventos operativos con timestamp (sin datos de impresoras) Rotación por tamaño Diagnóstico y auditoría de operación del servicio.
4. Configuración Cifrada (config.enc)

La configuración se almacena en C:\ProgramData\STCCloudMonitor\config.enc con cifrado AES-256-GCM. La clave se deriva con PBKDF2 (SHA-256, 100.000 iteraciones, salt único) a partir del Hardware ID del equipo. Ningún campo de configuración se transmite al servidor.

Campo Contenido Sensibilidad
serverUrl URL del portal STC Cloud Baja
agentId UUID asignado al agente en la activación Baja
token JWT de autenticación Bearer Alta
refreshToken Token de refresco JWT Alta
ipRanges Array de rangos IP a escanear (definidos por el administrador) Media
snmpCommunity Community string SNMP (típicamente "public") Media
snmpVersion Versión SNMP (1 ó 2c) Baja
scanIntervalMinutes Frecuencia de escaneo en minutos Baja
proxyUrl URL del proxy corporativo (opcional; puede incluir credenciales) Media

Hardware ID: La clave de cifrado se deriva del MachineGuid (registro de Windows) + Serial de BIOS, procesados con SHA-256. Este valor nunca se almacena en texto claro ni se transmite. Su único propósito es ligar la configuración al hardware físico del equipo, impidiendo su uso en otro dispositivo.

5. Datos Explícitamente NO Recopilados
Categoría Ejemplos Confirmación técnica
PII de usuarios Nombres, emails, DNI, cuentas de AD No existe código de acceso a AD/LDAP/SAM en el agente
Contenido de documentos Archivos impresos, texto de documentos SNMP solo expone contadores numéricos, no spools de impresión
Actividad de usuario Capturas de pantalla, historial de aplicaciones Agente corre como servicio de sistema sin acceso a sesiones de usuario
Contraseñas de red Credenciales de dominio, WiFi, otras aplicaciones No existe acceso al Credential Manager de Windows
Inventario de software Aplicaciones instaladas en el equipo host Solo accede a Registry para leer MachineGuid (derivación de clave)
Tráfico de red Contenido de paquetes, DNS queries, navegación Solo genera tráfico SNMP saliente a los rangos IP configurados por el admin

7. Clasificación de Riesgo de Privacidad
Vector de Riesgo Nivel Justificación
Exposición de PII Ninguno El agente no accede ni procesa datos de personas físicas
Exposición de datos de negocio sensibles Bajo Solo contadores numéricos de páginas — sin contenido de documentos
Exposición de topología de red Bajo Las IPs de impresoras son privadas (RFC 1918); los rangos de escaneo no se transmiten
Credenciales comprometidas Bajo JWT cifrado con AES-256-GCM ligado al Hardware ID — inutilizable en otro equipo
Pérdida de datos Bajo Retención local máxima 7 días; los contadores son datos replicables desde las impresoras
Intercepción en tránsito Bajo HTTPS obligatorio con TLS 1.2+; soporte de proxy corporativo configurable
8. Responsabilidades y Contacto
Rol Responsabilidad sobre los datos
Proveedor (STC) Custodio de los datos recibidos en el portal. Responsable de su almacenamiento, procesamiento y cumplimiento normativo en el servidor.
Cliente (IT Admin) Define los rangos IP de escaneo y administra el equipo host. Propietario de la base de datos local. Puede auditar logs en C:\ProgramData\STCCloudMonitor\agent.log.

Para auditores: Los logs de auditoría de integridad Ed25519 del agente registran cada actualización automática con timestamp ISO, versión y firma verificada. Accesible mediante stc-monitor --status (campo lastLog) o directamente en el archivo de log.
