1. Activación Silenciosa del Agente

El agente se activa mediante un único comando no interactivo. No se presentarán diálogos ni prompts al usuario. Todos los errores se reportan exclusivamente vía código de salida y mensaje en STDERR.

CMD / PowerShell — Activación
agente.exe --activate <CLAVE_DE_ACTIVACION> --url https://portal.empresa.com
PowerShell — Captura del resultado
$proc = Start-Process "agente.exe" -ArgumentList "--activate CLAVE --url https://portal.empresa.com" -Wait -PassThru -WindowStyle Hidden
Write-Host "Exit code: $($proc.ExitCode)"

2. Tabla de Códigos de Salida

El agente utiliza códigos de salida diferenciados para permitir que herramientas de despliegue automático (Inno Setup, Microsoft Intune, SCCM, PowerShell DSC) identifiquen con precisión la causa del fallo sin necesidad de parsear mensajes de texto.

Código Comando(s) Significado Acción Recomendada
0 — OK --activate, --status, --set-proxy Operación completada con éxito. El agente está operativo y en buen estado. Ninguna. Continuar flujo de instalación.
1 — ERROR GENÉRICO --activate, --status Error no clasificado: argumentos faltantes, error del servidor (HTTP 5xx), o en --status: agente activo pero sin conectividad al portal, o fallo de descifrado de configuración no relacionado con HWID. Revisar STDERR para el detalle. Verificar URL del portal y estado de red.
2 — CLAVE INVÁLIDA --activate El servidor rechazó la clave de activación (HTTP 400, 401 ó 403). La clave puede ser incorrecta, expirada, ya utilizada o no corresponder a este tenant. Verificar la clave en el portal STC Cloud. Generar una nueva clave de activación si expiró. No reintentar con la misma clave.
3 — ERROR DE RED --activate El agente no pudo establecer conexión TCP con el servidor portal. Causas: DNS no resuelve, firewall bloquea el puerto 443, proxy no configurado, o timeout de conexión. Verificar conectividad al portal desde el equipo objetivo. Revisar reglas de firewall (puerto 443 saliente). Si hay proxy corporativo, configurar previamente con --set-proxy.
Nota para Inno Setup: Los códigos 2 y 3 permiten mostrar mensajes de error específicos en el wizard del instalador usando ResultCode en la sección [Run]. Ver Sección 5 para el ejemplo completo.

3. Interfaz de Diagnóstico --status

El flag --status emite un objeto JSON en STDOUT con el estado completo del agente en el momento de la consulta. Está diseñado para ser consumido por scripts de monitoreo, herramientas MDM y dashboards de operaciones sin interpretación de logs de texto.

CMD / PowerShell — Consulta de estado
agente.exe --status

Código de Salida de --status

--status retorna exit 0 si el agente está en buen estado, o exit 1 si se detecta alguna de las siguientes condiciones críticas:

  • hardware_binding_integrity es "hwid-mismatch" — el HWID del equipo cambió.
  • hardware_binding_integrity es "decrypt-error" — la configuración está corrupta.
  • El agente está activado pero cloud_connectivity.reachable es false.
Un agente no activado ("activated": false) retorna exit 0 porque la falta de activación no es un error operativo: el equipo aún no ha sido enrolado. El sistema de inventario puede usar este estado para identificar equipos pendientes de activación.

Ejemplo de Respuesta — Estado Saludable

JSON — agente activo y en buen estado (exit 0)
{
  "version":                    "1.5.6",
  "timestamp":                  "2026-05-16T14:30:00.000Z",
  "activated":                  true,
  "agentId":                    "agt_a1b2c3d4",
  "serverUrl":                  "https://portal.empresa.com",
  "service_status":             "running",
  "cloud_connectivity": {
    "reachable":  true,
    "latencyMs": 142,
    "httpStatus": 200
  },
  "hardware_binding_integrity": "ok",
  "dataDir":                    "C:\\ProgramData\\STCCloudMonitor",
  "proxyUrl":                   null,
  "lastLog":                    "[2026-05-16T14:29:55Z] [INFO] Ciclo de escaneo completado. 24 dispositivos."
}

Ejemplo de Respuesta — HWID Mismatch (exit 1)

4. Esquema JSON de Respuesta de --status

Campo Tipo Descripción y valores posibles
version string Versión del binario del agente (ej. "1.5.6").
timestamp string (ISO 8601) Momento exacto UTC en que se ejecutó --status. Permite detectar resultados cacheados o desfasados.
activated boolean true si existe configuración cifrada válida y descifrable con el HWID actual.
agentId string | null Identificador único del agente en el portal. null si no está activado.
serverUrl string | null URL del portal STC Cloud configurado. null si no está activado.
service_status string Estado del servicio Windows STCCloudMonitor:
"running", "stopped", "starting", "unknown", "not-installed".
cloud_connectivity object Resultado del health check al endpoint /api/v1/health del portal (timeout 5s).
Campos: reachable (bool), latencyMs (número, solo si reachable), httpStatus (código HTTP, solo si hay respuesta), error (string, solo si falla). Valor especial de error: "not-activated" cuando el agente no tiene URL configurada.
hardware_binding_integrity string Estado del enlace criptográfico entre configuración y hardware:
ok — Config descifrada correctamente con HWID actual.
config-missing — No existe archivo de configuración (agente nunca activado).
hwid-mismatch — El HWID del equipo no coincide con el de activación (placa base o BIOS reemplazada). Requiere reactivación.
decrypt-error — Error al descifrar por causa distinta al HWID. Posible corrupción del archivo de configuración.
dataDir string Ruta del directorio de datos del agente (por defecto C:\ProgramData\STCCloudMonitor).
proxyUrl string | null URL del proxy HTTP configurado. null si conexión directa.
lastLog string | null Última línea del archivo agent.log. Útil para diagnóstico rápido sin acceso directo al equipo.

5. Integración con Inno Setup

El instalador Inno Setup puede capturar el código de salida de la activación y mostrar un mensaje de error específico al técnico de campo, o abortar silenciosamente la instalación con un log para análisis posterior.

Inno Setup Script (.iss) — Sección [Run] con manejo de exit codes
; Paso 1: Activar el agente (no interactivo)
[Run]
Filename: "{app}\agente.exe";
Parameters: "--activate {code:GetActivationKey} --url {code:GetServerUrl}";
StatusMsg: "Activando el agente STC Cloud...";
Flags: runhidden waituntilterminated;

; Paso 2: Verificar resultado en código Pascal
[Code]
var
  ActivationResult: Integer;

function NextButtonClick(CurPageID: Integer): Boolean;
begin
  Result := True;
  if CurPageID = wpFinished then
  begin
    { Leer exit code almacenado por el paso Run }
    case ActivationResult of
      0: { Éxito — no hacer nada };
      2: MsgBox('Error: La clave de activación es inválida o ha expirado.'#13#10
               'Solicite una nueva clave en el portal STC Cloud.',
               mbError, MB_OK);
      3: MsgBox('Error: No se pudo conectar al servidor STC Cloud.'#13#10
               'Verifique conectividad de red y reglas de firewall (puerto 443).',
               mbError, MB_OK);
    else
      MsgBox('Error de activación (' + IntToStr(ActivationResult) + ').'#13#10
             'Revise el log en C:\ProgramData\STCCloudMonitor\agent.log',
             mbError, MB_OK);
    end;
  end;
end;
Importante: El flag runhidden es obligatorio para garantizar que la activación sea completamente silenciosa. Sin él, el proceso podría mostrar una ventana de consola al usuario final.

Verificación Post-Instalación

Se recomienda agregar un paso adicional de verificación al final del instalador usando --status y comprobando que hardware_binding_integrity sea "ok":

PowerShell — Verificación post-instalación embebida en Inno Setup
$raw    = & "C:\Program Files\STCCloudMonitor\agente.exe" --status
$status = $raw | ConvertFrom-Json

if ($status.hardware_binding_integrity -ne 'ok') {
    Write-Error "HWID integrity: $($status.hardware_binding_integrity)"
    exit 1
}
Write-Host "Agent OK. ID: $($status.agentId)"
exit 0

6. Integración con Microsoft Intune / SCCM (PowerShell)

Los scripts de detección de Intune y SCCM pueden usar --status para verificar la salud del agente de forma estándar, sin acceder a archivos de log.

Script de Detección (Intune Win32 App)

PowerShell — Detection script para Intune (exit 0 = instalado y sano)
# Intune detection: exit 0 = app present and healthy, exit 1 = not detected
$agente = "C:\Program Files\STCCloudMonitor\agente.exe"

if (-not (Test-Path $agente)) { exit 1 }

try {
    $raw    = & $agente --status 2>$null
    $status = $raw | ConvertFrom-Json

    if ($status.activated -and
        $status.hardware_binding_integrity -eq 'ok' -and
        $status.service_status -eq 'running') {
        Write-Host "STC Agent OK: $($status.agentId)"
        exit 0
    }
    exit 1
} catch { exit 1 }

Script de Remediación (Intune)

PowerShell — Remediation script: reactivar si HWID mismatch
$agente  = "C:\Program Files\STCCloudMonitor\agente.exe"
$clave   = "<CLAVE_DESDE_SECRET_VAULT>"
$portal  = "https://portal.empresa.com"

$raw     = & $agente --status 2>$null
$status  = $raw | ConvertFrom-Json

if ($status.hardware_binding_integrity -eq 'hwid-mismatch') {
    # Hardware reemplazado: borrar config antigua y reactivar
    Remove-Item "C:\ProgramData\STCCloudMonitor\config.enc" -Force -ErrorAction SilentlyContinue
    & $agente --activate $clave --url $portal
    if ($LASTEXITCODE -ne 0) {
        Write-Error "Reactivacion fallida (exit $LASTEXITCODE)"
        exit 1
    }
}

Start-Service "STCCloudMonitor" -ErrorAction SilentlyContinue
exit 0
Seguridad de credenciales: Nunca almacene la clave de activación en el script en texto plano. Use Azure Key Vault, SCCM Secrets, o una variable de entorno protegida inyectada por Intune en tiempo de ejecución.

7. Interpretación de Errores de Seguridad en Logs

El agente registra eventos de seguridad con nivel [ERROR] en el archivo C:\ProgramData\STCCloudMonitor\agent.log. Los siguientes mensajes requieren atención inmediata por parte del equipo de seguridad:

Fragmento del mensaje Significado Severidad
VIOLACION DE INTEGRIDAD [Ed25519]: La firma del paquete de actualizacion NO es valida El paquete de auto-actualización descargado del servidor no tiene una firma Ed25519 válida. El agente rechazó la actualización. Posibles causas: paquete manipulado en tránsito (MITM), compromiso del servidor de actualizaciones, o archivo corrupto. CRÍTICO
VIOLACION DE INTEGRIDAD [Ed25519]: Error al verificar firma del paquete Error técnico al leer o procesar el archivo de firma (.sig). El agente rechazó la actualización por precaución. Puede ser un error transitorio de red o un problema en el servidor. ALTO
HWID_MISMATCH El Hardware ID del equipo ha cambiado desde la activación. La configuración cifrada no es descifrable con el hardware actual. El agente está inoperativo hasta que se reactive. MEDIO
Protocolo ante VIOLACION DE INTEGRIDAD [Ed25519]: Aísle el equipo de la red corporativa, notifique al equipo de seguridad, y audite los servidores de distribución de actualizaciones. Este evento no debe ocurrir en operación normal y puede indicar un ataque activo.

8. Procedimiento de Reactivación por HWID Mismatch

Este escenario ocurre cuando se reemplaza la placa base, la BIOS o se migra el sistema operativo a nuevo hardware. La configuración cifrada queda ligada al hardware antiguo y no puede descifrarse.

  1. Verificar el estado con agente.exe --status y confirmar "hardware_binding_integrity": "hwid-mismatch".
  2. Detener el servicio: net stop STCCloudMonitor
  3. Eliminar la configuración antigua: del "C:\ProgramData\STCCloudMonitor\config.enc"
  4. Generar una nueva clave de activación para este equipo en el portal STC Cloud (el agente anterior quedará marcado como inactivo).
  5. Reactivar: agente.exe --activate <NUEVA_CLAVE> --url https://portal.empresa.com
  6. Verificar el exit code (debe ser 0) e iniciar el servicio: net start STCCloudMonitor
  7. Confirmar con agente.exe --status que "hardware_binding_integrity" es "ok".
Backup de configuración: No se recomienda hacer backup del archivo config.enc ya que está criptográficamente ligado al Hardware ID del equipo original. Restaurarlo en hardware diferente resultará siempre en HWID mismatch.

9. Configuración de Confianza de Endpoint

El binario agente.exe es un ejecutable nativo de Windows empaquetado con Node.js mediante pkg. Esta técnica de empaquetado puede generar alertas en Windows SmartScreen o Windows Defender en equipos donde el software no haya sido visto previamente. Las siguientes secciones describen los procedimientos validados para garantizar una ejecución ininterrumpida en entornos corporativos.

9.1 Desbloqueo de SmartScreen (Instalador)

Si al ejecutar STC-Monitor-Setup.exe aparece la pantalla azul de SmartScreen:

  1. Hacer clic en "Más información" (enlace debajo del mensaje de advertencia).
  2. Hacer clic en el botón "Ejecutar de todas formas" que aparece.
  3. Continuar con la instalación normalmente.

Para desbloquear un archivo descargado desde el Explorador de Windows (sin ejecutarlo):

  1. Hacer clic derecho sobre STC-Monitor-Setup.exe → Propiedades.
  2. En la pestaña General, marcar la casilla "Desbloquear" en la sección de seguridad.
  3. Hacer clic en Aceptar y ejecutar el instalador.

O bien desde PowerShell (requiere privilegios de administrador):

Unblock-File -Path "C:\Descargas\STC-Monitor-Setup.exe"

9.2 Exclusión en Windows Defender (Post-instalación)

Para evitar que Windows Defender interrumpa la ejecución del agente como servicio, añadir una exclusión de ruta permanente. Ejecutar en PowerShell como Administrador: 

# Excluir el directorio de instalación del agente
Add-MpPreference -ExclusionPath "C:\Program Files\STC Cloud Monitor"

# Excluir también el directorio de datos (opcional, recomendado)
Add-MpPreference -ExclusionPath "C:\ProgramData\STCCloudMonitor"

# Verificar que las exclusiones quedaron registradas
Get-MpPreference | Select-Object -ExpandProperty ExclusionPath
Nota de seguridad: Las exclusiones de Windows Defender deben aplicarse únicamente a las rutas específicas del agente, nunca a unidades completas o directorios de sistema. Documentar la exclusión en el registro de cambios de seguridad de la organización.

9.3 Despliegue vía GPO (Exclusión Centralizada)

Para entornos con múltiples equipos, la exclusión puede distribuirse mediante Group Policy:

  1. Abrir Group Policy Management en el Domain Controller.
  2. Crear o editar una GPO vinculada a la OU con los equipos monitoreados.
  3. Navegar a: Computer Configuration → Administrative Templates → Windows Components → Microsoft Defender Antivirus → Exclusions → Path Exclusions.
  4. Añadir C:\Program Files\STC Cloud Monitor como valor de exclusión.
  5. Forzar actualización de política: gpupdate /force

9.4 Verificación de Integridad (SHA-256)

Antes de instalar, cualquier equipo de seguridad puede verificar que el binario descargado coincide exactamente con el publicado por STC Cloud, usando el script de verificación incluido: 

# Ejecutar desde PowerShell en el directorio de descarga
.\verify_hash.ps1 -FilePath "STC-Monitor-Setup.exe" -ExpectedHash "<HASH_OFICIAL>"

El script verify_hash.ps1 se distribuye junto al instalador y calcula el hash SHA-256 localmente sin enviar datos a ningún servidor externo.
